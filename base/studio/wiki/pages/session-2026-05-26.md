# Session Summary: 2026-05-26

## What Was Built

### 1. Planner V4 Pro Tuning (5 fixes)

- **V4 Pro for all phases** — plan generation + execution + replan 全用 Pro
- **max_tokens 16384** — Pro thinking tokens 需要大预算
- **Plan prompt 简化** — 长 example 让 Flash/Pro 都产出不完整，简化后 3 步全出
- **YAML 解析兼容** — 支持 `{steps: [...]}` 嵌套、`name` 代替 `id`、`changes` 代替 `instruction`
- **Flash vs Pro 对比**: Flash 2s/3步，Pro 70s/3步。两者质量可用，Pro 更详细

### 2. Skill Layer (`core/skills.py`)

Tool / Skill / Agent 三层能力体系：

```
Agent (dispatch / planner)
  ├── Skill（多步操作，带前置检查）
  │     ├── unity_control: pre_check, ensure_running, refresh, build_panel
  │     ├── screenshot_qa: capture, evaluate, capture_and_score
  │     └── feishu_report: send_text, send_image, send_grid
  └── Tool（原子操作）
        └── read_file, write_file, edit_file, bash
```

10 个 skill 注册并可用。dispatch 的 `run_qa_review` 和 `run_studio_report` 重构为调用 skill。

### 3. MCP HTTP 直连

- **去掉 npx 依赖** — `_unity_mcp_call` 从 `subprocess.run(["npx", ...])` 改为 `urllib.request` 直接 HTTP POST
- **UNITY_MCP_URL 可配置** — 环境变量 `UNITY_MCP_URL`，默认 `http://localhost:25891`
- **不再依赖 go-poo-client 的 MCP 配置** — 我们的系统自己管理连接

### 4. Unity/Tuanjie 自动启动

- `unity_control.ensure_running` 自动检测并启动 Tuanjie Editor
- 支持 `/Applications/Tuanjie/Tuanjie.app` + Unity Hub 两种路径
- 用 `open -a` + `-projectPath` 直接打开项目
- 实测：Tuanjie 10 秒内 MCP 可达

### 5. tool_runner 兼容性修复

- **path 参数变体** — DeepSeek V4 Pro 用 `filePath`/`file_path`/`file` 代替 `path`
- **reasoning_content 传回** — V4 Pro thinking mode 必须在多轮对话中传回 reasoning_content
- **model ID 修正** — `deepseek-chat` 是 V4 Flash 不是 Pro，改为 `deepseek-v4-pro`

### 6. 工作流纪律

- **确立原则**: 系统建设用 CLI，工作流执行必须走 dispatch
- 所有 GamePanel bug fix 重新通过 dispatch 任务执行
- Unity 状态检查集成到 skill 前置检查

## Key Discoveries

1. **`deepseek-chat` = V4 Flash** — 之前所有 dispatch 用的都是弱模型
2. **V4 Pro thinking 消耗大量 output tokens** — 50 max_tokens 产出空响应
3. **V4 Pro reasoning_content 必须传回** — 多轮 tool call 否则 400 错误
4. **Plan prompt 越简越好** — 长 example/rules 让 LLM 反而产出不完整
5. **skill pre_check 必须在执行前** — Unity 不在线直接拦截，不盲跑
6. **MCP HTTP 直连比 npx 快 10 倍+** — 0.05s vs 2s+ per call
7. **Tuanjie ≠ Unity** — 安装路径不同，需要兼容
8. **capture-panel 进入 Play Mode** — 截图后 Editor domain reload，MCP 暂时不可用
9. **GameScene 必须先打开** — 否则 UIManager 找不到，capture 只有天空盒

## What Failed

- **GamePanel 截图全是天空盒** — capture 前没有 open GameScene
- **Plan 生成只出 1 步** — V4 Pro thinking 消耗 + 长 prompt 导致截断（已修复）
- **tool_runner KeyError: 'path'** — V4 Pro 用不同参数名（已修复）
- **手动绕过 dispatch 修改代码** — 违反了工作流纪律（已纠正）
